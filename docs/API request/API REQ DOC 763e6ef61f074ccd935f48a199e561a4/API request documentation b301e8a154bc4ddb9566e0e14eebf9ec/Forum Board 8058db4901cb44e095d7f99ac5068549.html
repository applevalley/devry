게시판 목록 불러오기

  • url mapping: ('board/forum')
  • method: get
  • 헤더에 사용자 인증 토큰 첨부 여부 : O (로그인하지 않은 경우 토큰은 빈값으로 넘어감)
  • 설명: tag_filter에의해 걸러지고 tab에 의해서 정렬된 게시물들의 목록을 요청한다.
  • 추가 설명: Forum 게시판 불러올 게시물 목록
    1. 토큰으로 로그인한 사용자가 확인가능할 경우:

      ① 팔로우한 사람들의 피드 :

      • 일주일 이내로 팔로우 하는 사람들이 발행한 글들 가져옥 중 좋아요 갯수와 날짜가 빠른 순을 종합 평가하여 정렬된 게시물들

      ② 추천글 피드 :

      • 좋아요가 많은 글과 최신 글을 종합 평가하여 정렬된 게시물들 (무한 스크롤 형태이기 때문에, 팔로우 하는 사람 및 태그의 수가 적을 경우, 비어있는 상태보다 다른 글을을 이어서 표시하는게 좋다고 생각하기 때문)
      • 일주일 이내로 좋아요를 받이 받은 글들 가져오기 (좋아요 순또는 최신순으로 받아오면 더 좋음)

    2 . 토큰이 비어있는 경우 또는 토큰으로 사용자가 확인 불가능할 경우:

    • 일주일 이내로 좋아요를 받이 받은 글들 가져오기 (추천글 피드)

  • 전송할 데이터: 없음
  • 기대하는 response data:
// form
{
 //* feed_list: [  // api 요청 사용자가 확인이 될 경우 해당 사용자가 팔로우하는 사람들의 최신글 목록을 불러온다.
 //*		{
      post_id, // 포럼 글 번호
      title, // 글 제목
			written_time, // 글 작성 시간
      ref_tags, // 참조된 태그
			liked, // 현재 로그인한 유저의 해당 글에 대한 좋아요 상태 표시
      comment_num, // 댓글 갯수
      viewed_num, // 조회 수
      thumnail, // 글의 썸네일 이미지
      like_num, // 포럼 글 좋아요 갯수
      user_info: {
        // 글 작성자 정보
        user_id, // 유저 구분 가능한 id
        username, // 유저 닉네임
        profile_img,  // 글 작성자의 프로필 이미지
      },
 //*   },
 //* ],
  recommend_list: [  // api 요청 사용자 확인되는지 상관없이 좋아요추천을 많이 받은 글 목록을 불러온다.
          {
            post_id, // 포럼 글 번호
            title, // 글 제목
						written_time, // 글 작성 시간
            ref_tags, // 참조된 태그
						liked, // 현재 로그인한 유저의 해당 글에 대한 좋아요 상태 표시
            comment_num, // 댓글 갯수
            viewed_num, // 조회 수
            thumnail, // 글의 썸네일 이미지
            like_num, // 포럼 글 좋아요 갯수
            user_info: {
              // 글 작성자 정보
              user_id, // 유저 구분 가능한 id
              username, // 유저 닉네임
              profile_img,  // 글 작성자의 프로필 이미지
            },
          },
  ],
}
