follow

배정
상태수정 요구

용어 설명

  • 토큰 유저: 헤더에 토큰을 넣어서 api 요청을 하는 유저
  • 프로필 유저: 토큰 유저가 현재 보고 있는 프로필의 유저
  • 팔로워 (follower) : 프로필 유저를 팔로잉을 하는 사람
  • 팔로이 (followee): 프로필 유저가 팔로잉 하는 사람

상황 설명

토큰 유저가 프로필 유저의 프로필을 보고 팔로워 수 또는 팔로이 수를 누를 경우 구체적인 팔로워/팔로이 목록이 보이는 모달을 띄우낟.

팔로워 목록 불러오기

  • url mapping: ('follow/followers')
  • method: get
  • 헤더에 사용자 인증 토큰 첨부 여부 : O (로그인 하지 않은 경우 토큰에는 빈 값이 들어간다)
  • 설명 : 프로필 유저의 팔로워 목록 정보를 요청. 토큰을 통해 is_following 값이 결정된다.
  • 전송할 data : 
    // form
{
	user_id, // 프로필 유저의 아이디
}
    // example
{
	user_id: 3, // 프로필 유저의 아이디
}
  • 기대하는 response data :
    // form
{
    follower_list:
    [  // 팔로워들의 정보 list
       {
           user_id,              // 해당 팔로워를 구분해 줄 id
					 username,        // 해당 팔로워의 닉네임
           profile_img,     // 해당 팔로워의 프로필 이미지
           follower_num,    // 해당 팔로워가 가진 팔로워 수
           post_num,        // 해당  팔로워가 작성한 글의 갯수
           is_following,    // 토큰 유저가 해당 팔로워를 팔로잉하고 있는지 여부 
       },
    ]
}
    // example
{
		follower_list: [
	        {
	          user_id: 1,
	          username: '유저',
	          profile_img: 'https://cdn.quasar.dev/img/avatar.png',
	          follower_num: 50,
	          post_num: 10,
	          is_following: true,
	        },
					{
	          user_id: 2,
	          username: '유저2',
	          profile_img: 'https://cdn.quasar.dev/img/boy-avatar.png',
	          follower_num: 22,
	          post_num: 6,
	          is_following: false,
	        },
					// 이와 같이 유저 숫자 만큼 리스트 안에 담아서 보내주시면 됩니다.
		],
}

팔로잉

  • url mapping: ('/follow/:user_id')
  • method: put
  • 헤더에 사용자 인증 토큰 첨부 여부 : O
  • 설명: 토큰 유저가 user_id에 해당하는 사용자를 팔로우할 수 있도록 요청
  • 전송할 data : 없음
  • 기대하는 response data : 없음

언팔로잉

  • url mapping: ('/follow/:user_id')
  • method: delete
  • 헤더에 사용자 인증 토큰 첨부 여부 : O
  • 설명: 토큰 유저가 user_id에 해당하는 사용자를 언팔로우할 수 있도록 요청
  • 전송할 data : 없음
  • 기대하는 response data : 없음
